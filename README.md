The project aims to implement an RDBMS for a ‘Library Management System'. The primary objective of implementing such a database is to eliminate manual record keeping and maintain a relational database with structured schema, efficient security as role-based access control and ensuring normalization to reduce data redundancy. 
The tools used here are Oracle SQL Data Modular for Data flow diagram and ER model. DFD is used here to show how data moves from entities, processes and store the information's in data stores. This is relevance to ER model for creating entities, attributes, constraints and form relationship between entities. generating relational schema to ensure normalization. Another tool used here is SQLite for creating tables , inserting data as SQL queries helps for data retrieval and indexing query optimization that allows faster searches and large-scale operations.
The major outcomes can be all library operations are in one centralized platform , better record maintenance compared to traditional file system , reduce time and cost-saving and enhancing user experience through easily accessing the information.

Project Objectives:
Design an ER model for the selected business scenario.
Convert the ER model into a normalized relational model.
Implement tables, constraints, and sample data using SSMS / SQLite.
Write SQL queries for retrieval and data manipulation.
Demonstrate transactions, rollback, commit, backup, recovery, and storage concepts.
